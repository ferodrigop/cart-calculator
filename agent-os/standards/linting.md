# Linting & Formatting Standards (Ruff)

Ruff is the only Python linter and formatter for this project.

1. **Configure in `pyproject.toml`.** Set `line-length = 100`,
   `target-version = "py312"` (or the locked Python version), and `src = ["app", "tests"]`
   so first-party import sorting works correctly.
2. **Curated `lint.select`, never `ALL`.** Start with:
   ```toml
   [tool.ruff.lint]
   select = ["E", "F", "I", "B", "UP", "SIM", "C4", "PT", "RUF", "ASYNC", "S", "N"]
   ```
   Avoid `select = ["ALL"]` — it silently enables new rules on every Ruff upgrade.
3. **Use the Ruff formatter, not Black.** Configure:
   ```toml
   [tool.ruff.format]
   quote-style = "double"
   ```
   Run `ruff format` in CI. One tool, one config, one cache.
4. **Per-file ignores for tests and migrations.**
   ```toml
   [tool.ruff.lint.per-file-ignores]
   "tests/**" = ["S101", "S105", "S106"]
   "alembic/versions/**" = ["E501"]
   ```
   Asserts and hardcoded passwords are fine in tests; autogenerated migration lines run
   long.
5. **Pre-commit + CI.** Wire `astral-sh/ruff-pre-commit` hooks (`ruff check --fix`,
   `ruff format`). In CI, run `ruff check .` (no `--fix`) and `ruff format --check .` so
   PRs that bypass the hook fail loudly.
