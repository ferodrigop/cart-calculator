# Spec: frontend-spa

## Goal

Add a small but production-grade React SPA in `frontend/` that consumes the backend
auth + checkout API. Three pages: `/login`, `/register`, `/cart`. Root `/` redirects
based on auth state. The page renders the canonical breakdown card
(`subtotal / taxes / discount / total`) after a successful `POST /checkout`.

The scaffold from `2026-05-06-0000-scaffold-foundation` is the starting point ‚Äî
empty themed `App.tsx`, shadcn `components.json`, Tailwind theme tokens, `cn()`
helper, strict tsconfig, `@/*` alias.

## Out of scope

- History page (Phase 2 per `agent-os/product/roadmap.md`).
- Settings / profile pages.
- Backend changes ‚Äî frontend assumes the contract described below.
- Server-rendered routes / Next.js ‚Äî we are a Vite SPA.
- Multi-currency / i18n.

## API contract assumed

Per orchestrator brief:

- `POST /auth/register {email, password}` ‚Üí `201 {access_token, refresh_token}`
- `POST /auth/login {email, password}` ‚Üí `200 {access_token, refresh_token}`
- `GET /auth/me` (Bearer) ‚Üí `200 {id, email}`
- `POST /auth/refresh {refresh_token}` ‚Üí `200 {access_token, refresh_token}`
- `POST /checkout` (Bearer) `{items: [{name, unit_price, quantity}]}`
  ‚Üí `200 {subtotal, taxes, discount, total}`
- API base URL via `VITE_API_BASE_URL` (default `http://localhost`).

If backend diverges at merge, the orchestrator reconciles ‚Äî we do not attempt to
adapt to multiple shapes.

## Standards followed

- `@agent-os/standards/frontend/vite-react.md` ‚Äî strict TS, `@/*` alias, env vars
  via `import.meta.env.VITE_*`, single module-scoped `QueryClient`, typed query
  keys, `src/lib/api.ts` central client, feature-co-located hooks under
  `src/features/*`.
- `@agent-os/standards/frontend/shadcn-tailwind.md` ‚Äî install primitives via
  `npx shadcn@latest add`, theme via CSS variables (already in `index.css`),
  forms via `<Form>` + `react-hook-form` + `zod`, compose primitives, use
  `cn()`.

## Auth model

- **Access token:** held in module-scope memory inside `src/lib/auth.ts`. Never
  persisted ‚Äî survives navigation but not full reload (re-acquired via the
  refresh token on first 401 or on app boot if a refresh token exists).
- **Refresh token:** `sessionStorage["cc.refresh"]`. `sessionStorage` (not
  `localStorage`) so it dies with the tab ‚Äî minimal session footprint, in line
  with the project's "smallest credible" principle.
- **Interceptor:** `apiFetch()` injects `Authorization: Bearer <access>` when
  present. On `401`, it attempts a single refresh, retries the original
  request, and on refresh failure clears tokens and bounces to `/login`.
- **No `/auth/me` polling.** Auth state is "we hold a refresh token" + "the
  last call did not 401 us out" ‚Äî the refresh round-trip is the source of truth.

## Routing

`react-router-dom` v6.

- `/` ‚Äî `<RootRedirect>`: `isAuthenticated()` ‚Üí redirect to `/cart`, else to
  `/login`.
- `/login` ‚Äî public. On success, `navigate("/cart", { replace: true })`.
- `/register` ‚Äî public. On success, same as login.
- `/cart` ‚Äî guarded by `<RequireAuth>`. Bounces to `/login` if no refresh token.

## Server state

- Single `QueryClient` at module scope in `src/lib/queryClient.ts`.
- Query key factories in `src/lib/queryKeys.ts` (`authKeys.me()`,
  `checkoutKeys.last()`).
- Mutations: `useLogin`, `useRegister`, `useCheckout`. Each in its feature
  folder.

## Forms

- `src/features/auth/loginSchema.ts` ‚Äî `email().min(1)`, `password.min(8)`.
- `src/features/auth/registerSchema.ts` ‚Äî same shape.
- `src/features/cart/cartSchema.ts` ‚Äî `items: array({name.min(1),
  unit_price.positive(), quantity.int().positive()}).min(1)`.

## Deliverables

### `frontend/package.json`

Add runtime deps: `react-router-dom`, `@tanstack/react-query`,
`react-hook-form`, `@hookform/resolvers`, `zod`, `lucide-react`, `sonner`,
plus the shadcn primitive deps (`@radix-ui/react-label`, `@radix-ui/react-slot`).

### `frontend/src/components/ui/`

Generated by shadcn CLI: `button`, `input`, `label`, `form`, `card`, `table`,
`sonner`.

### `frontend/src/lib/`

- `api.ts` ‚Äî typed `apiFetch<T>(path, init)` wrapper around `fetch`. Bearer
  injection, single-flight 401 refresh, JSON-only request/response handling,
  surface non-2xx as `ApiError`.
- `auth.ts` ‚Äî `getAccessToken / setAccessToken / getRefreshToken /
  setRefreshToken / clearTokens / isAuthenticated`. Listeners so the router
  can re-render on logout.
- `queryClient.ts` ‚Äî module-scope `QueryClient` with sensible defaults
  (no refetch on window focus for the cart).
- `queryKeys.ts` ‚Äî typed key factories.

### `frontend/src/features/auth/`

- `loginSchema.ts`, `registerSchema.ts`.
- `useLogin.ts`, `useRegister.ts` ‚Äî TanStack mutations that POST to the API
  and store tokens on success.
- `LoginPage.tsx`, `RegisterPage.tsx` ‚Äî shadcn `<Form>` with email + password
  fields, error toast on failure, link to the other route.

### `frontend/src/features/cart/`

- `cartSchema.ts`.
- `useCheckout.ts` ‚Äî mutation posting `{items}` and returning the breakdown.
- `CartPage.tsx` ‚Äî header (email + log out), items table with add row,
  remove row, edit name/price/qty inline; client-side subtotal label;
  `Submit checkout` button ‚Üí renders `<CheckoutSummary>` card on success.
- `CheckoutSummary.tsx` ‚Äî shadcn `<Card>` listing the four totals. Currency
  formatted via `Intl.NumberFormat("en-US", { style: "currency", currency:
  "USD" })`.

### `frontend/src/routes.tsx`

`createBrowserRouter` with the four routes + `<RootRedirect>` and
`<RequireAuth>`.

### `frontend/src/App.tsx`

Replace the placeholder with `<QueryClientProvider>` + `<RouterProvider>` +
`<Toaster richColors />`.

### `frontend/src/vite-env.d.ts`

Type `VITE_API_BASE_URL` on `ImportMetaEnv`.

### `frontend/.env.example`

`VITE_API_BASE_URL=http://localhost`

## Acceptance

- `cd frontend && npx tsc --noEmit` ‚Äî clean.
- `cd frontend && npm run build` ‚Äî succeeds.
- `cd frontend && npm run dev` ‚Äî boots; `/` renders without console errors and
  redirects to `/login`.
- (Post-merge, orchestrator) `docker compose up` then visit `http://localhost`
  ‚Üí register ‚Üí login ‚Üí cart ‚Üí checkout works end-to-end.
